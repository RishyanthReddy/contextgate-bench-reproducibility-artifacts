# Supplementary Table S27: Metric Definition Audit

| Definition group | Quantity | Operational definition | Computation or scope | Interpretation boundary | Source artifact or code |
| --- | --- | --- | --- | --- | --- |
| Input normalization | x_i expression vector | Processed nonnegative platform matrix with standardized gene symbols and aligned unit metadata; task features apply the registered task-layer transforms rather than pooling raw CosMx and Xenium scales directly. | LR/module and marker summaries use log1p or unit-interval summaries where registered; label centroid features use log1p plus row L2 normalization; CellPack rows use rank/order and expression-bin tokens from the same matrix. | Cross-platform summaries aggregate route/utility metrics after task-level normalization; they are not claims that raw platform count depths are interchangeable. | manuscript Section 3.3; src/cellpack/evaluation/*; src/cellpack/sequence_builder.py |
| Utility and regret | u_im and r_im | u_im is the registered route-label or benchmark-matrix metric utility mapped to [0,1]; r_im equals the in-setting empirical oracle utility minus u_im. | Route-level utilities come from the registered route utility matrix summarized in Supplementary Table S34; CP-Q6 metric rows use the normalized/proxy metric_value fields generated by the benchmark matrix. | The oracle is the best registered method in the evaluated set, not an absolute biological or algorithmic performance ceiling; CP-Q2 route regret measures label alignment. | Supplementary Table S34; src/cellpack/method_comparison.py; results/reports/cp_q6_benchmark_matrix/benchmark_metric_matrix.csv |
| Effect-control evidence gate | a_i_eff and source q-values | Binary registered effect-control flag used in Equation 5. For split-level rows, the source artifacts may include BH-adjusted q-values for their own context-vs-reference contrasts; for synthetic rows, the flag is the registered control-consistency gate. | ContextGate reads the source flag as fdr_effect_gate_passed. A numeric q-value, when present, belongs to the source comparison table and is not reinterpreted as a one-sided test of the aggregate Delta_i sign gate. | The effect-control flag can pass when the residual-utility sign gate fails because the two gates are different registered evidence checks; context_allowed requires both. Within-split BH screens are nominal and do not control the aggregate 40/63 pass count or spatially dependent tests globally. | src/cellpack/evaluation/router_target_mining.py; results/reports/cp_q6_benchmark_matrix/benchmark_metric_matrix.csv |
| Safety metrics | all-row false-context exposure | Fraction of all evaluated rows, regardless of null status, where a method uses context on a row labeled null, wrong-context, or unsupported. | Uses all rows as the denominator, unlike Equation 3 FCR, whose denominator is restricted to n_i = 1 rows. | This is a population-level exposure descriptor and need not be the complement of denominator-aware no-harm unless both are computed on the same row set. | Supplementary Table S24; results/reports/cp_q6_benchmark_matrix/benchmark_metric_matrix.csv |
| Safety metrics | FCR, no-harm, specificity | False-context rate is context use among null or unsupported rows; no-harm is one minus that rate; null specificity is non-context routing among synthetic null/control ladder rows. | Pooled and denominator-stratified calculations are reported separately so wrong-context, unsupported real-data, and synthetic-null rows are not silently conflated. | Expression-only and ContextGate can share FCR=0; ContextGate's added value is safety plus positive-control detection and auditable abstention. | Supplementary Table S24; Supplementary Table S35; results/reports/cp_q2_method_comparison/method_predictions.csv; results/reports/cp_q3_positive_null_control_ladder/positive_control_ladder.csv |
| Positive/null ladder | sensitivity score and specificity score | Continuous diagnostic scores stored in the CP-Q3 sensitivity and specificity columns and summarized as row means with normal-approximation mean intervals. | The positive-control sensitivity score is averaged over the 36 target-observable positive rows; the null/control specificity score is averaged over the 36 registered negative/control rows and repeated without panel-blocked rows in Supplementary Table S35. | These score means are distinct from the binary route recovery/rejection proportions in Equation 4; for ContextGate, 0.8633 and 0.9433 are score means, while 36/36 and 36/36 are binary route outcomes. | Supplementary Tables S19 and S35; results/reports/cp_q3_positive_null_control_ladder/positive_control_ladder.csv; results/reports/cp_q3_positive_null_control_ladder/method_summary.csv |
| Positive/null ladder | gate_pass_rate | Mean of the per-row gate_passed indicator in the synthetic positive/null ladder for a method family. | Positive rows pass when sensitivity clears the registered floor and route is context-positive; negative rows pass when specificity clears the floor, no false context is used, and route is negative or abstained. | This is control-consistency over synthetic labels, not the same quantity as sensitivity, specificity, or execution completeness. | src/cellpack/positive_control_ladder.py; results/reports/cp_q3_positive_null_control_ladder/method_summary.csv |
| Route audit scalar | confidence | Deterministic route-assurance scalar emitted by each registered method/router rule and averaged descriptively in route summaries. | ContextGate emits 0.95 for strict or negative routes and 0.85 for abstentions; method-family comparators emit fixed scalars attached to their registered route rules. | Confidence is not a calibrated posterior probability and is not used as the FDR test or as an independent biological claim. | src/cellpack/contextgate.py; src/cellpack/method_comparison.py; results/reports/cp_q5_contextgate_decisions/contextgate_decisions.csv |
| Aggregate operating point | FCR and sensitivity/specificity thresholds | Publication-facing policy thresholds: FCR <= 0.05, route recovery >= 0.95, sensitivity >= 0.65, and specificity >= 0.80. | Thresholds are checked at the aggregate ContextGate decision level and stress-tested against moderate and permissive operating points. | These are safety-first claim-release rules, not universal biological constants; exploratory users may relax them with explicit false-context exposure. | Supplementary Table S26; src/cellpack/contextgate.py; src/cellpack/robustness_stress.py |
| Registration | registered artifact | A dataset, method, threshold, schema, command, or result table materialized in repository manifests and artifact registries before manuscript claim release. | Registry entries link artifact paths, generating commands, schema expectations, benchmark phases, and validation checks under version control. | Registered here means version-controlled benchmark registration; it is distinct from clinical-trial preregistration or an external timestamped DOI until release deposition. | data/manifests/contextgate_benchmark_manifest.yaml; data/manifests/artifact_registry.yaml; manuscript/tables/manuscript_table_manifest.json |
